# Test Automation Framework Learning

这是一个基于开源项目进行学习与实践的接口自动化测试框架练习仓库，主要用于系统学习 Python + Pytest 接口自动化测试 的框架结构、环境搭建、Mock 服务联调、测试执行与问题排查过程。

我在学习过程中完成了 Windows 本地环境下的项目部署、依赖安装、Pytest 版本兼容处理、Mock 服务启动、测试执行与结果分析，并对自动化测试框架的分层设计与工程化思路进行了实践与总结。

---

## 项目定位

本项目不是从零独立原创开发的测试框架，而是基于优秀开源项目进行学习、搭建、调试和实践的个人仓库。

## 项目结构理解

本项目采用了较为典型的接口自动化测试框架分层设计，主要目录含义如下：

### `testcase`
存放测试用例，包含单接口测试、商品管理相关测试以及业务场景测试。

### `common`
存放公共工具方法，例如断言封装、连接封装、日志记录等。

### `base`
存放基础能力封装，例如请求基类、接口调用能力等。

### `conf`
存放配置相关内容，用于实现测试逻辑与环境参数分离。

### `data`
存放测试数据，支撑参数化测试和数据驱动执行。

### `report`
存放测试执行结果与测试报告相关内容。

### `mock_server`
存放本地模拟服务，用于在无真实后端接口时完成联调与测试执行。
